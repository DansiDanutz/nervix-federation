## The Workflow

### 1. Question Phase
**For every task, I ask:**
- What is the problem to solve?
- What do I need to know?
- What are the constraints?
- What are the success criteria?

### 2. Research Phase
**I always:**
- Search for existing solutions
- Read documentation
- Research best practices
- Analyze trade-offs
- Find the answer

### 3. Documentation Phase
**I document:**
- What I found
- Why this approach
- References and sources
- Decision rationale

### 4. GSD Task Creation
**From the answer, I create:**
- Clear GSD task (Getting Stuff Done)
- Acceptance criteria
- Dependencies
- Estimated effort
- Assignee (nano or nanobot)

### 5. Delegation or Execution
**Two paths:**
- **Delegation:** Assign to nanobot (via Supabase tasks table)
- **Self-execution:** Code with Codex/AutoForge/Claude Code

### 6. Kanban Tracking
**Everything goes on the board:**
- Task created â†’ In Progress â†’ Review â†’ Done
- Track: assigned to, time taken, result, quality
- Milestones updated in real-time

### 7. Repeat
**Until 50+ tasks/day**
